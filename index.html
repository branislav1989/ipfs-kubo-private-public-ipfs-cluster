Step 1: Get Your API Credentials

Generate your unique API credentials with a single command. Save these credentials in a secure password manager immediately.

curl -X POST https://api.datahosting.company/register

This is the only time your `api_secret`, `dashboard_token`, and `ipfs_access_hash` will be shown.

{
  "api_key": "YOUR_UNIQUE_API_KEY",
  "api_secret": "YOUR_UNIQUE_API_SECRET",
  "dashboard_token": "YOUR_UNIQUE_DASHBOARD_TOKEN",
  "ipfs_access_hash": "YOUR_UNIQUE_IPFS_ACCESS_HASH"
}

Note: The `ipfs_access_hash` is used internally to prevent unauthorized access to your files. Keep it secure along with your other credentials.

How to Pin a File

curl -X POST https://api.datahosting.company/api/pins \
     -H "X-API-KEY: YOUR_API_KEY" \
     -H "X-API-SECRET: YOUR_API_SECRET" \
     -F "file=@/path/to/your/file.jpg" \
     -F "retention_months=1" \
     -F "private=true"

How to Create a Backup

curl -X POST https://api.datahosting.company/api/cluster/backup \
     -H "X-API-KEY: YOUR_API_KEY" \
     -H "X-API-SECRET: YOUR_API_SECRET" \
     -F "file=@/path/to/your/archive.zip" \
     -F "replica_count=2" \
     -F "retention_days=365"

Note: retention_days is required. Choose how many days to store (e.g., 30 for 1 month, 365 for 1 year, 730 for 2 years)
